# ADR-001: Ingestão de Dados

**Status:** Aceito **Data:** 06/03/2026 **Autores** Guilherme Lima, Matheus Araújo e Yohann

## 1. Contexto

Existe a necessidade de extrair diariamente dados do ERP (vendas, estoque, financeiro e logística), garantindo a atualização com defasagem máxima de **24 horas**.

A solução deve atender aos seguintes requisitos:

* *Execução automaizada:* Sem intevenção manual.
* *Baixo custo operacional:* Otimização de recursos financeiros.
* *Escalabilidade:* Suporte ao crescimento do volume de dados.
* *Facilidade de manutenção:* Código modular e simples.
* *Integração Azure:* Compatibilidade nativa com o ecossistema em nuvem.
* *Severless:* Evitar o gerenciamento de infraestrutura dedicada.

## 2. Decisão

Decidimos utilizar o *Azure Functions* no modelo severless, utilizando o gatilho de agendamento, para realizar a ingestão automática dos dados do ERP.

As funções serao responsáveis por:
1. Conectar via API/Database ao ERP.
2. Extrair os dados incrementais do periodo.
3. Persistir os dados brutos no *Azure blobl Storage*.

## 3. Consequências

(+) *Redução de custo:* Pagamento apenas pelo tempo de execução.
(+) *Alta escalabilidade:* Escalonamento automático baseado na carga.
(+) *Manutenção simplificada:* Foco total no código, sem preocupação com patches de SO.

(-) *Vendo lock-in:* Forte dependencia da plataforma Azure. 
(-) *Controle limitado:* Menor controle sobre o ambiente de execeução e limites de timeout.

## 4. Alternativas Rejeitadas

* *Máquinas vituais:* Rejeitadas devido ao maior custo e necessidade de manutenção contínua de infraestrutura.
* *Containers:* Rejeitado pela complexidade operacional excessiva para o escopo atual.
* *Scripts locais/Execução manual:* Rejeitados por não atenderem aos requisitos de automação, disponibilidade e confiabilidade
