*# ADR-002: EStratégia de Armazenamento

> *Status:* Aceito
> *Data:* 06/03/2026
> *Autores:* Guilherme Lima, Matheus Araújo e Yohann

## 1. Contexto

Após a ingestão e transformação dos dados provenientes do ERP, é necessário amrazená-los de forma estruturada para suportar consultas analíticas e alimentar dashboards no Power BI.

A solução deve atender aos seguintes critérios: 
* *Consistência:* Garantir a integridade dos dados transacionais.
* *Performance:* Permitir consultas eficientes para grandes volumes.
* *Single Source of Truth:* Servir como fonte única de verdade para a organização.
* *Interoperabilidade:* Integração nativa com ferramentas de BI.
* *Cloud Native:* Operar integralmente em ambiente de nuvem.

## 2. Decisão

Decidimos utilizar o *Azure SQL Database* como o repositório principal para o armazenamento dos dados tratados e normalizados (Gold Layer/Data Warehouse).

O banco de dados será configurado para:
* Armazenar dados estruturados seguindo modelagem dimensional (Star Schema).
* Suportar a camada de consumo analítico.
* Servir como fonte primária para os datasets do *Power BI*.

## 3. Consequências

* (+) *Performance:* Excelente desempenho em consultas relacionais e analíticas.
* (+) *Integração:* Conexão nativa e simplificada com o ecossistema Microsoft (Power BI, Synapse, Data Factory).
* (+) *PaaS (Platform as a Service):* Redução drástica da carga administrativa (backups, patches e tuning automáticos).
* (+) *Segurança:* Recursos avançados de criptografia e controle de acesso (RBAC).

* (-) *Custo:* Modelo de cobrança recorrente baseado em DTUs ou vCores.
* (-) *Modelagem:* Exige rigor na definição de esquemas e índices para manter a performance.

## 4. Alternativas Rejeitadas

* *NoSQL (ex: Cosmos DB):* Rejeitado por não ser otimizado para consultas analíticas complexas e agregações estruturadas.
* *Blob Storage (Data Lake puro):* Rejeitado por não oferecer a performance de indexação e a facilidade de consulta direta via SQL para os dashboards.
* *Banco On-premise:* Rejeitado por ir contra a estratégia de transformação digital e escalabilidade em nuvem.
